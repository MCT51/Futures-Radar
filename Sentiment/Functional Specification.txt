A general investigation of the problem and its background
Typically, futures work happens speculatively, oscillating between interpretation without transparent evidence and working with large volumes of data difficult to interpret and extract meaning from. The Education Futures Radar is an attempt to create a practical, accessible, evidence-informed infrastructure that supports futures-oriented thinking in education. It focuses on questions about where signals of change are emerging, how developments interact, and where uncertainty or disagreement remains. This aligns closely with the concept of futures literacy: the capacity to use ideas about the future to inform decisions in the present. The Education Futures Radar should support this by making futures-relevant evidence easier to collect, structure, and explore, supporting decision-making across educational contexts. This includes horizon scanning and strategy for policy makers, and informing teachers on professional orientation within broader social and technological shifts.
The Functionality to be Provided
The main goal of our project is identifying, auditing, and structuring futures-relevant data. The aim is a working prototype that demonstrates the feasibility of the Education Futures Radar and enables at least one form of futures-informed exploration that was not previously possible. We are focusing on a working proof of concept that demonstrates automated data ingestion, structured storage, and basic visual explorations of signals.
The major components of the system
1. Automatic collection and updating of signals from identified and verified data sources
2. A clean and documented data schema for signals 
3. A working visual dashboard connected to the live dataset
interface specifications for those components
1.  
We will produce an automated data ingestion pipeline  that has the following functionalities:
a.                     Pull data from selected sources
b.                    Runs on a repeatable schedule
c.                     Handles basic cleaning and normalisation
d.                    Store outputs in a consistent, structured format.
The pipeline must be reproducible and extendable, with transparent transformations.
The pipeline input will be data from audited data sources. The data will be from a range of sources and in a range of formats, eg RSS feeds, CSV files etc.
The output of the pipeline will be CSV files
2.  
Our data pipeline will output CSV files of ‘signals of change’. We will develop a minimal but meaningful schema for these signals. The schema must be consistent, extensible, and able to support multiple visualisation techniques. We will also produce documentation to describe the schema.
3.  
We will produce a web app that displays exploratory visualisations to demonstrate the value of our structured signals data. The exact method of visualisation will be chosen later depending on the sources used. Possible methods include timelines, maps, or comparisons. The visualisations must be clearly connected to the underlying data and quick to adapt as data sources change.
The visual dashboard will be connected to the live dataset. It will be developed in python, using existing dashboard tools.
 
acceptance criteria for the finished product
The project will be considered successful if we deliver:
-           A data audit of candidate sources
-           A working, automated data ingestion pipeline
-           A structured, populated signals dataset
-           At least 1 live, visual exploration of the data
-           Clear documentation and next steps recommendations
Meeting the goals that:
a)       New data can be added with minimal additional work
b)      The dataset can support meaningful, exploratory questions
c)       DEFI can see clear pathways to further development.
a management strategy for the group
We will use a shared GitHub repository to manage our code. The group will meet regularly on Tuesdays and Thursdays to monitor progress and discuss the project.
We have a shared google drive with the clients to allow them to share helpful information and resources with us. We also have a WhatsApp group with the clients for easy and quick communication.
